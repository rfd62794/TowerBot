# ADR-016: Dual Gmail Account Pattern

## Status
Accepted

## Context
PrivyBot needs to monitor Gmail for two distinct accounts:
1. **Personal Gmail** — User's personal email for daily monitoring
2. **RFD IT Services Gmail** — Professional email for IT work notifications

Both accounts require OAuth2 authentication via Google Cloud Console. The challenge is:
- Each account needs its own OAuth token file
- Some tools need to query both accounts (merged inbox summary)
- Some tools need to query a specific account (search_email)
- RFD credentials may not be available in all environments (dev vs production)
- Tools must gracefully handle missing RFD credentials

## Decision
Implement a dual-account pattern with per-account credential files and graceful degradation.

### Credential Storage
```
config/
  youtube_token.json    # Personal account (reused for Gmail, Calendar, Tasks)
  rfd_token.json        # RFD IT Services account (optional)
```

### API Client Pattern
```python
# tools/api/gmail_api.py

def _get_credentials(token_file: str = "youtube_token.json") -> Credentials | None:
    """Load OAuth credentials from specified token file."""
    token_path = os.path.join(os.path.dirname(__file__), "../config", token_file)
    if not os.path.exists(token_path):
        return None
    return Credentials.from_authorized_user_file(token_path, SCOPES)

def _build_service(credentials: Credentials) -> Resource:
    """Build Gmail API service from credentials."""
    return build("gmail", "v1", credentials=credentials)

# Personal account (always available)
def get_personal_inbox_summary() -> dict:
    """Get inbox summary for personal account."""
    creds = _get_credentials("youtube_token.json")
    if not creds:
        return {"error": "Personal credentials not found"}
    service = _build_service(creds)
    # ... fetch and return summary

# RFD account (may be missing)
def get_rfd_inbox_summary() -> dict:
    """Get inbox summary for RFD account."""
    creds = _get_credentials("rfd_token.json")
    if not creds:
        return {"error": "RFD credentials not found"}
    service = _build_service(creds)
    # ... fetch and return summary
```

### Tool Layer Pattern
```python
# tools/gmail.py

def get_inbox_summary() -> dict:
    """Merged inbox summary from both accounts."""
    from tools.api.gmail_api import get_personal_inbox_summary, get_rfd_inbox_summary

    personal = get_personal_inbox_summary()
    rfd = get_rfd_inbox_summary()

    result = {
        "personal": personal if "error" not in personal else None,
        "rfd": rfd if "error" not in rfd else None,
    }

    # If both available, merge counts
    if result["personal"] and result["rfd"]:
        result["total_unread"] = result["personal"]["unread"] + result["rfd"]["unread"]
    elif result["personal"]:
        result["total_unread"] = result["personal"]["unread"]
    elif result["rfd"]:
        result["total_unread"] = result["rfd"]["unread"]
    else:
        result["total_unread"] = 0

    return result

def search_email(query: str, account: str = "personal") -> dict:
    """Search emails in specific account."""
    from tools.api.gmail_api import search_personal_email, search_rfd_email

    if account == "personal":
        return search_personal_email(query)
    elif account == "rfd":
        return search_rfd_email(query)
    else:
        return {"error": f"Unknown account: {account}"}
```

### Graceful Degradation
- RFD token file missing → API functions return `{"error": "RFD credentials not found"}`
- Tool layer checks for error and returns `None` for that account
- Merged summaries work with partial data (personal only)
- Tools that require RFD explicitly fail if RFD missing

## Consequences
### Positive
- Clear separation of credentials per account
- Personal account always available (same token as YouTube)
- RFD account optional (dev environments can skip)
- Merged inbox works with partial data
- Easy to add more accounts in future

### Negative
- More credential files to manage
- Token reuse (YouTube token used for Gmail/Calendar/Tasks) — coupling
- Tools need to handle missing credentials explicitly

### Mitigations
- Document credential file purposes in .env.example
- Use descriptive token file names (youtube_token.json for personal)
- Add health check at startup to warn if RFD missing in production
- Keep credential loading centralized in API layer

## Examples

### Personal account only (dev environment)
```python
# config/rfd_token.json does not exist
result = get_inbox_summary()
# Returns: {"personal": {...}, "rfd": None, "total_unread": 5}
```

### Both accounts available (production)
```python
# Both token files exist
result = get_inbox_summary()
# Returns: {"personal": {...}, "rfd": {...}, "total_unread": 12}
```

### Search specific account
```python
result = search_email("invoice", account="rfd")
# Returns RFD search results or error if RFD missing
```
