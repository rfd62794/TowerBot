# ADR-025: think Scratchpad Tool

## Status
Accepted

## Context
The agent had no visible reasoning channel before executing complex multi-step tasks.

**Problem:**
- Complex multi-step tasks lacked planning
- No record of agent's reasoning process
- Model switches lost reasoning context
- When DeepSeek throttles and Llama picks up, the new model has no record of what was being worked on
- No way to track the agent's thought process

**Example Scenario:**
1. User asks: "Analyze my top 5 videos and tell me which retention curve needs improvement"
2. Agent calls get_top_videos() → gets list
3. DeepSeek throttles (429 error)
4. Model switches to Llama
5. Llama has no context about the analysis plan
6. Llama might start over or lose track of the original goal

## Decision
Create `think` tool as a simple function in `tools/meta/meta.py` — no BaseTool, no API calls, no caching.

### Architecture
```python
# tools/meta/meta.py

def think(thought: str) -> dict:
    """
    Record a reasoning step before acting.
    Creates a visible scratchpad entry.

    Provides context continuity when
    models switch due to throttling —
    the next model sees the thought
    and can continue from it.

    Returns immediately. No side effects.
    """
    return {
        "ok": True,
        "thought": thought,
        "stale_notice": None
    }
```

### Tool Registry Integration
```python
# tools/__init__.py

TOOL_REGISTRY = {
    "think": {
        "fn": think,
        "definition": {
            "type": "function",
            "function": {
                "name": "think",
                "description": "WHEN: Before a complex tool chain where multiple steps are needed. When the question requires planning before executing. When switching between topics or when resuming after a tool failure.\n\nWHY: Creates visible context that persists across model switches. If one model throttles and another takes over, the thought record lets the new model continue from where the previous model left off.\n\nRETURNS: thought recorded, ok=True. No side effects. No storage.\n\nDO NOT CALL: for every single message. Use for genuinely complex multi-step reasoning only.\n\nEXAMPLE: think('I need to find the top video first, then check its retention curve to see where viewers drop.') → then call get_top_videos() → then call get_retention_curve()",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "thought": {
                            "type": "string",
                            "description": "Your reasoning step or plan",
                        },
                    },
                    "required": ["thought"],
                },
            },
        },
    },
}
```

### Report Integration
```python
# bot/agent.py

async def _execute(thread_id: str, name: str, args: dict) -> dict:
    if name in TOOL_REGISTRY:
        tool_fn = TOOL_REGISTRY[name]["fn"]
        r = tool_fn(**args)
        
        if name == "think":
            await report("thought", thought=args.get("thought", ""))
        elif name == "save_commitment":
            await report("commitment_saved", ...)
        else:
            await report("tool_called", tool_name=name)
        
        return r
```

### Report Event Type
```python
# bot/report.py

def _format(event_type: str, kwargs: dict):
    if event_type == "thought":
        thought = kwargs.get("thought", "")
        if not thought:
            return None
        return f"💭 {thought}"
    return None
```

### System Prompt Rule
```python
# bot/agent.py

system_prompt = """
...
THINK BEFORE COMPLEX ACTIONS:
For questions requiring multiple tool calls or multi-step reasoning:
  1. Call think() with your plan first
  2. Execute the plan step by step
  3. Call think() again if direction changes

For simple single-tool questions:
  Skip think() — answer directly.

think() output is visible to you and to Robert — use it honestly.
"""
```

## Rationale
**Why not BaseTool:**
- BaseTool exists for API-calling tools with caching
- think has no network, no cache, no stale data
- Forcing BaseTool adds noise without benefit
- Simple function is more appropriate

**Why no storage:**
- Conversation history is sufficient
- No need for database writes
- Keeps think() lightweight
- Model-agnostic — any model sees past thoughts via conversation

**Why report("thought"):**
- Visible to Robert in Telegram
- Transparency into agent reasoning
- 💭 emoji distinguishes from other events
- No storage overhead

**Context Continuity Benefit:**
- think() creates conversation history record
- Model-agnostic — any model sees past thoughts
- When DeepSeek throttles and Llama picks up, the new model reads thoughts and continues the plan
- Throttled model switches preserve context

## Consequences
### Positive
- Agent reasoning visible to Robert
- Context preserved across model switches
- Complex tasks more reliably completed
- No storage overhead — conversation only
- Model-agnostic continuity

### Negative
- Additional tool call overhead
- Agent might overuse think() for simple tasks
- Adds noise to conversation history

### Mitigations
- System prompt instructs when to use think()
- Tool definition explicitly says "DO NOT CALL for every single message"
- Report event is visible but not intrusive
- Conversation history is already large — think() adds minimal overhead

## Example Usage
```
User: "Analyze my top 5 videos and tell me which retention curve needs improvement"

Agent: think("I need to get the top 5 videos first, then get retention curves for each, then compare drop-off points.")
→ think() returns {"ok": True, "thought": "..."}
→ report("thought") fires 💭 in Telegram

Agent: get_top_videos(limit=5)
→ returns list of 5 videos

Agent: get_retention_curve(video_id="...")
→ returns retention data

[DeepSeek throttles, model switches to Llama]

Llama reads conversation history:
- User request
- think("I need to get the top 5 videos first, then get retention curves for each, then compare drop-off points.")
- get_top_videos() result
- get_retention_curve() result

Llama continues: get_retention_curve(video_id="...")
→ continues from where DeepSeek left off
```
