## ADR-032: mem0 Integration

**Status:** Planned

**Context:**
PrivyBot's memory system has two gaps. Search quality: LIKE-based substring matching is query-sensitive — `get_memories("ReactReel")` works, `get_memories("projects goals ReactReel")` returns zero. Knowledge depth: 27 manually seeded facts vs hundreds of hours of Claude and Gemini conversations containing dense context about every project, decision, and architectural choice. That corpus is currently inaccessible.

Ollama is already installed on the Nitro laptop. Tower receives Ollama during Phase 7 deployment. Fully local mem0 is achievable now, no Tower required to start.

**Decision:**

**Package:** `mem0ai` — `Memory` class (self-hosted), not `MemoryClient` (mem0.ai platform). No data reaches mem0's servers.

**Telemetry:** `MEM0_TELEMETRY=false` — explicitly disabled in environment.

**Embedding model:** `nomic-embed-text` via Ollama.
- `ollama pull nomic-embed-text` 
- 137M parameters, ~274MB
- Outperforms OpenAI text-embedding-ada-002 on short direct queries — which is the memory retrieval use case
- Fast on Nitro GPU, acceptable on Tower i5 CPU (single query embed ~50ms)
- Same pull command on both machines

**Extraction model:** `gemma3:4b` via Ollama.
- `ollama pull gemma3:4b` 
- 128K context window handles long conversation export chunks
- Explicitly designed for low-resource deployment, strong at classification and extraction
- ~2.5GB disk, ~3-4GB RAM at inference
- Slow on Tower CPU for batch pre-dump — acceptable (one-time operation, not real-time)

**Vector store:** Chroma with SQLite backend.
- `pip install chromadb` 
- No Docker, no server, pure Python, persistent on disk
- Same store on Nitro and Tower

**Zero external API calls** for all memory operations. Embedding, extraction, and retrieval are fully local.

**What mem0 replaces:** `get_memories()` semantic queries only. The existing SQLite memory table is retained — layer-based filtering, manual seeds, agent save/update/retire all unchanged. mem0 adds semantic search alongside the structured system, not instead of it.

**What mem0 does NOT touch:** goals, tasks, commitments, cache, history tables. Unrelated.

**System prompt integration:**
```python
# _system_prompt() — combined results:
semantic   = memory_manager.search(recent_context, limit=5)
structured = list_memories()   # existing, unchanged
# combined → injected into system prompt
```

**Pre-dump pipeline:**
```
Source:   Claude conversation exports (JSON via claude.ai data export)
          Gemini exports (Google Takeout JSON)

Script:   scripts/pre_dump_memories.py
          Reads export files
          Chunks into conversation segments
          Calls mem0.add(messages, user_id="robert") per chunk
          gemma3:4b handles extraction
          mem0 deduplicates by content hash
          Re-runnable safely

One-time operation. Can be re-run as new exports become available.
```

**Migration of existing 27 memories:**
```
Script:   scripts/migrate_memories_to_mem0.py
          Reads SQLite memory table
          Adds each to mem0 with metadata (layer, key, source="seed")
          SQLite table unchanged
```

**Configuration:**
```
OLLAMA_BASE_URL=http://localhost:11434   # default
MEM0_TELEMETRY=false
MEM0_EMBEDDING_MODEL=nomic-embed-text
MEM0_EXTRACTION_MODEL=gemma3:4b
```

**Phase ordering:**
Phase 15 begins on Nitro immediately (Ollama present). Tower receives Ollama as part of Phase 7 deployment — mem0 picks it up automatically. Phase 9 (background task models) proceeds independently.

**Implementation order:**
1. `pip install mem0ai chromadb httpx` 
2. `ollama pull nomic-embed-text && ollama pull gemma3:4b` 
3. `infra/memory_manager.py` — MemoryManager wrapping mem0
4. `scripts/migrate_memories_to_mem0.py` 
5. `scripts/pre_dump_memories.py` 
6. Update `bot/agent.py` — `_system_prompt()` combined results
7. Update `get_memories` tool routing through MemoryManager
8. Tests

**Consequences:**

*Positive:* Semantic search closes query sensitivity gap. Pre-dump corpus changes the knowledge base from 27 facts to thousands. Fully local — zero external API calls for memory operations. No new infrastructure (Chroma + Ollama, no Docker).

*Negative:* gemma3:4b adds ~3-4GB RAM on Tower during pre-dump (32GB headroom is comfortable). Pre-dump is manual per export source. Chroma is a new dependency.

*Mitigations:* Pre-dump script re-runnable, mem0 deduplicates. Chroma is pure Python, well-maintained. Tower RAM is not the constraint.
