# ADR-028: Tool and API Auto-Discovery

## Status
Planned

## Context

### Current State
- `tools/registry.py` requires manual editing for every new tool
- No canonical source for API client inventory
- `/status` command shows rate limits but not which API clients exist
- Tool definitions split between `tools/__init__.py` and `core/memory.py` (historical accident)
- No category or source metadata on tools
- `TOOL_INCOMPATIBLE` set in `model_manager.py` excludes models leaking raw tool-call text

### Problems
1. **Manual registry maintenance:** Adding a new tool requires editing `tools/registry.py`, updating imports, and running verify
2. **No tool organization:** Tools lack category metadata (content, productivity, search, etc.)
3. **No API inventory:** No central list of which API clients exist (Gmail, YouTube, Steam, etc.)
4. **No context-aware tool selection:** Agent cannot filter tools by category to reduce token count
5. **Scattered tool definitions:** Historical split between `tools/__init__.py` and `core/memory.py` caused circular imports
6. **Model filtering via exclusion:** `TOOL_INCOMPATIBLE` is a negative list rather than per-model metadata

## Decision

### Solution
Implement auto-discovery for tools and API clients using decorator-based registration and runtime scanning.

### Architecture
```
tools/_tool.py — @tool decorator
tools/_discover.py — discover_tools(), discover_apis()
tools/registry.py — populated automatically at boot
tools/api/_handler.py — BaseAPIHandler with SOURCE metadata
```

### Design

#### Tool Decorator
```python
# tools/_tool.py

from functools import wraps
from typing import Callable

def tool(name: str,
         description: str,
         parameters: dict,
         category: str = "general",
         source: str = None):
    """
    Decorator that marks a function as a registered tool.
    
    Usage:
    @tool(
        name="get_youtube_stats",
        description="Call when user asks about channel performance...",
        parameters={...},
        category="content",
        source="youtube"
    )
    def get_channel_summary(days: int = 7):
        ...
    """
    def decorator(fn: Callable) -> Callable:
        fn._tool_definition = {
            "type": "function",
            "function": {
                "name": name,
                "description": description,
                "parameters": parameters
            }
        }
        fn._tool_name = name
        fn._tool_category = category
        fn._tool_source = source
        fn._is_tool = True
        
        @wraps(fn)
        def wrapper(*args, **kwargs):
            return fn(*args, **kwargs)
        
        wrapper._tool_definition = fn._tool_definition
        wrapper._tool_name = name
        wrapper._tool_category = category
        wrapper._tool_source = source
        wrapper._is_tool = True
        
        return wrapper
    return decorator
```

#### Tool Discovery
```python
# tools/_discover.py

import importlib
import pkgutil
from pathlib import Path

def discover_tools(package_path: str) -> dict:
    """
    Scan a package directory for tool functions.
    Any function decorated with @tool is registered.
    """
    registry = {}
    
    for finder, name, ispkg in pkgutil.walk_packages(
            path=[package_path],
            prefix="tools."):
        module = importlib.import_module(name)
        
        for attr_name in dir(module):
            attr = getattr(module, attr_name)
            if hasattr(attr, "_is_tool"):
                registry[attr._tool_name] = {
                    "fn": attr,
                    "definition": attr._tool_definition,
                    "category": attr._tool_category,
                    "source": getattr(attr, "_tool_source", None)
                }
    
    return registry
```

#### API Discovery
```python
# tools/api/_discover.py

def discover_apis(package_path: str) -> dict:
    """
    Scan api/ directory for BaseAPIHandler subclasses.
    Any class inheriting BaseAPIHandler is registered.
    """
    from tools.api._handler import BaseAPIHandler
    import inspect
    
    api_registry = {}
    
    for finder, name, ispkg in pkgutil.walk_packages(
            path=[package_path]):
        module = importlib.import_module(name)
        
        for attr_name in dir(module):
            attr = getattr(module, attr_name)
            if (inspect.isclass(attr) and
                    issubclass(attr, BaseAPIHandler) and
                    attr is not BaseAPIHandler and
                    hasattr(attr, "CACHE_PREFIX")):
                
                api_registry[attr.CACHE_PREFIX] = {
                    "class": attr,
                    "instance": attr(),
                    "source": getattr(attr, "SOURCE", None),
                    "rate_limits": rate_limits.LIMITS.get(attr.CACHE_PREFIX)
                }
    
    return api_registry
```

#### Registry Population
```python
# tools/registry.py

from tools._discover import discover_tools, discover_apis

# Auto-discover all tools
TOOL_REGISTRY = discover_tools("tools/")

# Auto-discover all APIs
API_REGISTRY = discover_apis("tools/api/")

# Filter by category
CONTENT_TOOLS = {
    k: v for k, v in TOOL_REGISTRY.items()
    if v["category"] == "content"
}

PRODUCTIVITY_TOOLS = {
    k: v for k, v in TOOL_REGISTRY.items()
    if v["category"] == "productivity"
}

# Filter by source
GOOGLE_TOOLS = {
    k: v for k, v in TOOL_REGISTRY.items()
    if v["source"] == "google"
}

# Agent gets all tools
ALL_TOOLS = [v["definition"] for v in TOOL_REGISTRY.values()]

# Context-aware tool selection
def tools_for_category(category: str):
    return [
        v["definition"]
        for v in TOOL_REGISTRY.values()
        if v["category"] == category
    ]
```

### Tool Categories
- `content` — YouTube, games, content recommendations
- `productivity` — Gmail, calendar, tasks, personal tasks
- `search` — Web search, news, wiki, Reddit, weather, fetch
- `memory` — Save/update/retire/get memories
- `goals` — Goals, tasks, commitments
- `meta` — Think tool

### API Sources
- `google` — Gmail, Calendar, Tasks
- `youtube` — YouTube API
- `steam` — Steam, SteamSpy, ITAD
- `web` — DuckDuckGo, Wikipedia, Reddit, fetch

## Implementation Timing

### During Directory Restructure Session
- Every tool file will be touched during the move from `tools/api/` → `api/`
- Add `@tool` decorator to each tool function in the same pass
- Zero extra files modified beyond the restructure
- One pass for both directory move and decorator addition

### After Directory Restructure
1. Create `tools/_tool.py` with `@tool` decorator
2. Create `tools/_discover.py` with `discover_tools()` and `discover_apis()`
3. Update `tools/registry.py` to use auto-discovery
4. Add `SOURCE` metadata to `BaseAPIHandler` subclasses
5. Update `core/agent.py` to use category-filtered tools when appropriate
6. Add `/tools` command to list tools by category
7. Add `/apis` command to list API clients
8. Update `/status` to include API_REGISTRY status
9. Replace `TOOL_INCOMPATIBLE` with per-model tool filtering via metadata
10. Verify all tests pass

## Consequences

### Positive
- New tools require no registry editing — drop file, restart bot, tool is live
- Category metadata enables context-aware tool selection (reduces token count)
- API_REGISTRY enables automatic `/apis` command and health monitoring
- Tool definitions centralized in one canonical location (registry.py)
- No more circular import risks (registry is root, nothing imports back up)
- Per-model tool filtering via metadata instead of exclusion list
- Better tool organization for maintenance

### Negative
- Adds complexity: decorator pattern and runtime scanning
- Requires updating all existing tool files with decorator
- Runtime scanning adds startup overhead (minimal, one-time at boot)
- Requires category and source metadata discipline

### Mitigations
- Decorator added during directory restructure — one pass, not two
- Startup overhead is one-time at bot boot, negligible
- Category and source metadata are optional (defaults to "general"/None)
- Fallback to manual registry if auto-discovery fails
