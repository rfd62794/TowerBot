# ADR-017: Google OAuth Scope Management

## Status
Accepted

## Context
PrivyBot integrates with multiple Google APIs:
- YouTube Analytics API (existing)
- YouTube Data API (existing)
- Gmail API (new)
- Google Calendar API (new)
- Google Tasks API (new)

Each API requires specific OAuth2 scopes. The challenge is:
- Adding new scopes requires re-authentication (user must re-run OAuth flow)
- Token reuse: YouTube token used for Gmail/Calendar/Tasks (single personal account)
- Scope creep: Adding too many scopes increases security surface
- Need to balance functionality with minimal scope requests

## Decision
Use a single OAuth token per account with a consolidated scope list. Re-auth required when adding new scopes.

### Scope List (Personal Account)
```python
SCOPES = [
    # YouTube
    "https://www.googleapis.com/auth/yt-analytics.readonly",
    "https://www.googleapis.com/auth/youtube.readonly",
    # Gmail
    "https://www.googleapis.com/auth/gmail.readonly",
    # Calendar
    "https://www.googleapis.com/auth/calendar.readonly",
    # Tasks
    "https://www.googleapis.com/auth/tasks.readonly",
]
```

### Token Reuse Strategy
- Single token file (`youtube_token.json`) for all personal Google APIs
- Token contains all scopes requested during OAuth flow
- API clients share the same credential loading function
- When adding new scope: delete token file, re-run OAuth flow

### Credential Loading
```python
# tools/api/youtube_api.py (shared across all Google APIs)

SCOPES = [
    "https://www.googleapis.com/auth/yt-analytics.readonly",
    "https://www.googleapis.com/auth/youtube.readonly",
    "https://www.googleapis.com/auth/gmail.readonly",
    "https://www.googleapis.com/auth/calendar.readonly",
    "https://www.googleapis.com/auth/tasks.readonly",
]

def _get_credentials(token_file: str = "youtube_token.json") -> Credentials:
    """Load OAuth credentials with all required scopes."""
    token_path = os.path.join(os.path.dirname(__file__), "../config", token_file)
    creds = None

    if os.path.exists(token_path):
        creds = Credentials.from_authorized_user_file(token_path, SCOPES)

    if not creds or not creds.valid:
        if creds and creds.expired and creds.refresh_token:
            creds.refresh(Request())
        else:
            flow = InstalledAppFlow.from_client_secrets_file(
                CLIENT_SECRETS_FILE, SCOPES
            )
            creds = flow.run_local_server(port=0)
        with open(token_path, "w") as token:
            token.write(creds)

    return creds
```

### Re-authentication Process
When adding a new scope:
1. Update `SCOPES` list in API client
2. Delete existing token file: `rm config/youtube_token.json`
3. Restart bot: `uv run python privybot.py`
4. Bot detects missing token, initiates OAuth flow
5. User completes OAuth in browser
6. New token saved with updated scopes

### Scope Justification
| API | Scope | Justification |
|-----|-------|---------------|
| YouTube Analytics | `yt-analytics.readonly` | Read channel metrics |
| YouTube Data | `youtube.readonly` | Read video metadata, search |
| Gmail | `gmail.readonly` | Read emails, search inbox |
| Calendar | `calendar.readonly` | Read events, schedule |
| Tasks | `tasks.readonly` | Read task lists, sync |

All scopes are `.readonly` — no write access. This minimizes security risk.

## Consequences
### Positive
- Single token per account (simpler credential management)
- Re-auth only when adding new scopes (infrequent)
- All scopes readonly (minimal security risk)
- Shared credential loading across all Google APIs

### Negative
- Re-auth required when adding new scope (user friction)
- Token coupling: YouTube token used for unrelated APIs
- Scope creep risk if not careful about adding new scopes

### Mitigations
- Document scope additions in ROADMAP.md
- Use readonly scopes only
- Minimize scope requests (only what's needed)
- Add health check at startup to warn if token missing
- Document re-auth process in README

## Examples

### Adding Gmail API (initial implementation)
```python
# Before: SCOPES had YouTube only
SCOPES = [
    "https://www.googleapis.com/auth/yt-analytics.readonly",
    "https://www.googleapis.com/auth/youtube.readonly",
]

# After: Add Gmail scope
SCOPES = [
    "https://www.googleapis.com/auth/yt-analytics.readonly",
    "https://www.googleapis.com/auth/youtube.readonly",
    "https://www.googleapis.com/auth/gmail.readonly",  # NEW
]

# User must delete token and re-auth
```

### Re-authentication flow
```bash
# 1. Delete old token
rm config/youtube_token.json

# 2. Start bot
uv run python privybot.py

# 3. Bot detects missing token, prints:
# "No valid credentials found. Starting OAuth flow..."
# "Please visit this URL to authorize: https://accounts.google.com/..."

# 4. User completes OAuth in browser

# 5. Bot saves new token with updated scopes
# "Credentials saved to config/youtube_token.json"
```
