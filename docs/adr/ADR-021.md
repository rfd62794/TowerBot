# ADR-021: CacheManager Layer

## Status
Accepted

## Context
Cache behavior is currently scattered across multiple locations:
- `core/db/cache.py` — storage functions (cache_tool_result, get_cached_tool_result, get_stale_cached_result)
- `tools/api/_base.py` — call pattern (cached_api_call, stale_notice, make_params_hash)
- Individual tool files — TTL values hardcoded in each file
- `scripts/preload.py` — warm-up logic (not yet implemented)
- ADR-019 — staleness budgets documented but not in code

Problems:
- No single owner of cache behavior
- TTL values in code don't match ADR-019 documentation
- No invalidation mechanism
- No canonical TTL registry
- Continuing Phase 2 with `_base.py` means every API file needs a second migration pass when CacheManager arrives

## Decision
Create `core/cache.py` — CacheManager class as the single owner of all cache behavior.

### Architecture
```
Layer 6 — Tool Logic (tools/*.py)
  Owns: business logic, scoring, verdicts, result shaping, staleness unwrapping
  Does NOT: call APIs directly, manage cache

Layer 7 — Cache Manager (core/cache.py)
  Owns: ALL cache behavior
    - Fresh hit detection
    - Stale fallback logic
    - TTL policy per tool
    - Preload coordination
    - Cache invalidation
    - preload_log writes
  Does NOT: call APIs, know about tools

Layer 8 — API Clients (tools/api/*.py)
  Owns: raw HTTP calls only
    - Authentication
    - Request formation
    - Response parsing
    - Error raising (typed)
  Does NOT: cache, apply business logic
  Returns: raw parsed response dict
```

### Implementation
```python
# core/cache.py

class CacheManager:
    # Canonical TTL registry — source of truth
    TTL = {
        "youtube_channel": 86400,  # 24h
        "youtube_videos": 21600,  # 6h
        "gmail_personal": 300,    # 5min
        "gmail_rfd": 300,        # 5min
        "calendar_today": 900,   # 15min
        "google_tasks": 300,     # 5min
        "steam_library": 86400,  # 24h
        "steam_metrics": 7200,   # 2h
        "steamspy": 14400,       # 4h
        "itad_prices": 7200,     # 2h
        "weather": 3600,         # 1h
        "ddg_search": 1800,      # 30min
        "wikipedia": 86400,      # 24h
        "reddit": 900,           # 15min
    }

    # Staleness budget — acceptable stale age
    STALE_BUDGET = {
        "gmail_personal": 300,
        "gmail_rfd": 300,
        "calendar_today": 900,
        "google_tasks": 300,
        "weather": 3600,
        "ddg_search": 1800,
        "reddit": 900,
        "youtube_channel": 86400,
        "steam_library": 86400,
        "wikipedia": 86400,
        "steamspy": 14400,
        "itad_prices": 7200,
        "steam_metrics": 7200,
        "youtube_videos": 21600,
        "calendar_upcoming": 900,
    }

    def call(self, key: str, params_hash: str, live_fn: callable, stale_ok: bool = True) -> dict:
        """Main entry point for all API calls."""

    def get(self, key: str, params_hash: str) -> dict | None:
        """Fresh cache hit only."""

    def get_or_stale(self, key: str, params_hash: str) -> dict | None:
        """Fresh first, stale if expired."""

    def set(self, key: str, params_hash: str, data: dict) -> None:
        """Store with TTL from TTL policy."""

    def stale_notice(self, result: dict) -> str | None:
        """Human-readable staleness string."""

    def preload(self, tasks: list) -> dict:
        """Warm cache from task list."""

    def status(self) -> list[dict]:
        """Health of all cached tools."""

    def invalidate(self, key: str, params_hash: str = None) -> None:
        """Clear cached data."""

    def hash(self, *args, **kwargs) -> str:
        """Deterministic params hash."""

# Singleton — import this everywhere
cache = CacheManager()
```

### Usage Pattern
```python
# tools/api/weather_api.py
from core.cache import cache

def get_current_weather() -> dict:
    def _live():
        # HTTP call, no cache logic
        return {"temp_f": ..., ...}

    return cache.call("weather", cache.hash(), _live)
```

### Migration Path
1. Create `core/cache.py` with CacheManager class
2. Migrate `weather_api.py` to use `cache.call()`
3. Migrate remaining API files one by one
4. Deprecate `tools/api/_base.py` after all migrations complete
5. `core/db/cache.py` becomes internal (called only by CacheManager)

## Rationale for Class over Module
- **Clean namespace** — `cache.call()` is unambiguous. Module-level `call()` conflicts with generic naming.
- **Testability** — Can inject mock `CacheManager` in tests without touching global state.
- **Encapsulation** — TTL and stale_budget are internal to the class, not exposed module globals.
- **Extensibility** — Future cache strategies (LRU eviction, multi-tier cache) can be added as methods without breaking imports.
- **Consistent with layer pattern** — Other layers (Agent, Router, Scheduler) are classes. CacheManager as a class fits the architecture.

## Consequences
### Positive
- Single TTL source of truth
- Cache behavior testable in isolation
- Invalidation possible
- Phase 2 API files use `cache.call()` directly
- No double-migration needed
- Staleness budgets in code match ADR-019

### Negative
- New file to maintain
- Initial migration effort for existing API files
- Singleton pattern has tradeoffs

### Mitigations
- Migrate API files incrementally (one per Phase 2 file)
- Keep `_base.py` until all migrations complete
- Document migration pattern in ADR
- Add comprehensive test coverage
