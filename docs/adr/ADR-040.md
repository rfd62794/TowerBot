# ADR-040: Playwright Browser Automation Architecture

## Status: Accepted

## Context
PrivyBot can find opportunities and draft actions but cannot execute them on websites without an API. Many platforms (itch.io, YouTube Studio for certain actions, etc.) lack comprehensive APIs or require browser-based interactions.

## Decision
Playwright runs via sync API within the PrivyBot process. Each tool call launches its own browser context with a saved storageState profile, executes the action, and closes the context. No persistent browser process.

- **API:** Sync (not async) for simplicity
- **Browser:** Chromium headless for production, headed for profile setup only
- **Profile storage:** `config/playwright_profiles/{site}.json` — Playwright storageState format
- **Profile management:** Gitignored directory, manual transfer to Tower
- **Context lifecycle:** Per-call — launch → execute → close

## Consequences

### Positive
- **Simpler architecture:** No persistent browser process to manage or monitor
- **Safer:** No state leaks between calls — each action starts fresh
- **Easier debugging:** Each call is isolated — failures don't affect subsequent calls
- **Profile portability:** storageState files can be copied between machines

### Negative
- **Slower:** Browser launch overhead on each call (acceptable for occasional automation)
- **Profile setup friction:** Manual headed-mode login required for each site
- **No session persistence:** Cannot maintain long-lived sessions across calls

### Future Considerations
A separate `PrivybotPlaywright` NSSM service can be added later for performance if needed. This would maintain a persistent browser process and reuse contexts, at the cost of increased complexity and state management.

## Profile Setup Flow
1. Run `setup_profile("site_name")` from a Python shell on Tower
2. Browser opens in headed mode
3. Log in manually to the target site
4. Press Enter — session saved to `config/playwright_profiles/{site_name}.json`
5. Copy profile JSON to Tower (gitignored — manual transfer required)

## Sites Initially Supported
- itch.io (devlog posting, game page reading)
- YouTube Studio (comment pinning — actions not available via API)
