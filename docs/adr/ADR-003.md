# ADR-003: Caching Strategy

## Status
Accepted

## Context
PrivyBot makes frequent API calls to external services (YouTube, Steam, SteamSpy, IsThereAnyDeal). Without caching:
- Repeated queries hit rate limits unnecessarily
- API costs accumulate for paid services
- Response latency is higher for repeated data
- Some APIs (YouTube Analytics) have strict quotas

A caching layer is needed to reduce API calls while ensuring data freshness.

## Decision
Use SQLite-based caching with per-tool TTL (time-to-live) values.

### Cache Storage
**Table: `tool_cache`**
```sql
CREATE TABLE tool_cache (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    tool_name TEXT NOT NULL,
    params_hash TEXT NOT NULL,
    result TEXT NOT NULL,  -- JSON-encoded result
    fetched_at TEXT NOT NULL,  -- ISO 8601 timestamp
    expires_at TEXT NOT NULL,  -- ISO 8601 timestamp
    UNIQUE(tool_name, params_hash)
);

CREATE INDEX idx_tool_cache_lookup ON tool_cache(tool_name, params_hash, expires_at);
```

**Cache Key:**
- `tool_name`: Name of the tool function (e.g., "get_channel_summary")
- `params_hash`: MD5 hash of sorted JSON parameters

### TTL Values by Tool Type

**YouTube Analytics (6 hours):**
- `get_channel_summary` — Channel metrics change slowly
- `get_top_videos` — Video rankings change slowly
- `get_video_analytics` — Per-video metrics change slowly

**Steam/SteamSpy (24 hours):**
- `get_game_metrics` — Player counts and owners change daily
- `get_steamspy_data` — SteamSpy data updates daily

**IsThereAnyDeal (1 hour):**
- `get_sale_info` — Prices change frequently, need freshness

**Content Recommendations (12 hours):**
- `get_content_recommendations` — Content signals change moderately

**Installed Games (1 hour):**
- `get_installed_games` — Library changes infrequently but should be fresh

### Cache Functions

**Write:**
```python
def cache_tool_result(tool_name: str, params_hash: str, result: dict, ttl_hours: float) -> None:
    """Cache a tool result with TTL."""
    expires_at = (datetime.now() + timedelta(hours=ttl_hours)).isoformat()
    _exec(
        "INSERT OR REPLACE INTO tool_cache (tool_name, params_hash, result, expires_at) VALUES (?, ?, ?, ?)",
        (tool_name, params_hash, json.dumps(result), expires_at),
        commit=True
    )
```

**Read:**
```python
def get_cached_tool_result(tool_name: str, params_hash: str) -> dict | None:
    """Get cached result if not expired."""
    row = _exec(
        "SELECT result FROM tool_cache WHERE tool_name = ? AND params_hash = ? AND expires_at > datetime('now')",
        (tool_name, params_hash)
    ).fetchone()
    return json.loads(row["result"]) if row else None
```

### Cache Pattern

**Tool implementation:**
```python
def tool_function(params: dict) -> dict:
    # 1. Compute cache key
    params_hash = _hash_params(params)
    
    # 2. Check cache
    cached = get_cached_tool_result("tool_function", params_hash)
    if cached:
        return cached
    
    # 3. Fetch fresh data
    result = api_call(params)
    
    # 4. Cache result
    cache_tool_result("tool_function", params_hash, result, ttl_hours=TTL)
    
    return result
```

### Cache Bypass

**When to bypass cache:**
- User explicitly requests fresh data (future feature)
- Cache returns error (re-fetch)
- Tool is marked as "no-cache" (future feature)

**Manual cache invalidation:**
```python
# Delete expired entries (run periodically)
_exec("DELETE FROM tool_cache WHERE expires_at < datetime('now')", commit=True)

# Delete specific tool cache
_exec("DELETE FROM tool_cache WHERE tool_name = ?", (tool_name,), commit=True)
```

## Consequences
### Positive
- Reduced API calls — repeated queries hit cache
- Faster response times — cache reads are local
- Rate limit protection — fewer API hits
- Cost savings — less API usage for paid services
- Persistent across restarts — SQLite survives bot restarts

### Negative
- Stale data risk — cache may return outdated data
- Storage growth — cache table grows over time
- Cache invalidation complexity — need to handle TTL expiration
- Debugging difficulty — cache may mask API issues

### Mitigations
- Conservative TTL values — err on side of freshness
- Periodic cleanup — delete expired entries daily
- Cache metadata — include `fetched_at` in response for transparency
- Cache bypass option — allow force-refresh in future
- Monitoring — track cache hit/miss ratio

## TTL Rationale

**YouTube Analytics (6 hours):**
- Channel metrics (views, watch time) update daily
- Video rankings change slowly
- YouTube API quota is strict (10,000 points/day)
- 6 hours balances freshness with quota protection

**Steam/SteamSpy (24 hours):**
- SteamSpy updates daily
- Player counts change slowly
- No strict rate limits but unnecessary load
- 24 hours matches data update frequency

**IsThereAnyDeal (1 hour):**
- Prices change frequently during sales
- Historical lows don't change often
- ITAD has generous rate limits (1000/5min)
- 1 hour ensures sale prices are current

**Content Recommendations (12 hours):**
- YouTube upload counts change moderately
- Content gap signals are relatively stable
- 12 hours balances freshness with performance

**Installed Games (1 hour):**
- Library changes infrequently
- Should be relatively fresh for UX
- 1 hour is conservative for user-facing data

## Examples

### Cached Tool Implementation
```python
from core.db import cache_tool_result, get_cached_tool_result
import hashlib
import json

def _hash_params(params: dict) -> str:
    return hashlib.md5(json.dumps(params, sort_keys=True).encode()).hexdigest()

def get_channel_summary(days: int = 7) -> dict:
    """Get YouTube channel performance with caching."""
    # Check cache
    params = {"days": days}
    params_hash = _hash_params(params)
    cached = get_cached_tool_result("get_channel_summary", params_hash)
    if cached:
        return cached
    
    # Fetch fresh data
    result = youtube_api.get_channel_data(days)
    
    # Cache result (6 hour TTL)
    cache_tool_result("get_channel_summary", params_hash, result, ttl_hours=6)
    
    return result
```

### Cache Response Format
```python
# Cached response includes metadata
{
    "views": 1529,
    "watch_time_minutes": 128.5,
    "subscribers_gained": 7,
    "_cached": True,  # Indicates cached response
    "_fetched_at": "2026-05-29T14:00:00Z",  # When cached
    "_expires_at": "2026-05-29T20:00:00Z"  # When expires
}
```
