# ADR-005: Model Routing Strategy

## Status
Accepted

## Context
PrivyBot needs to call LLM models for AI responses. Key constraints:
- Account has $0 paid credits — must use free models
- Free models frequently return 429 (rate limit) errors from upstream providers (Venice, Crucible)
- Need tool-calling capability for memory functions
- Want access to multiple models for different use cases (default, deep thinking, Claude)

Initial approach would be to hardcode a single model, but this fails when:
- The hardcoded model is throttled (429)
- The hardcoded model goes offline
- Different use cases need different models (fast vs deep thinking)

## Decision
Use OpenRouter with dynamic model discovery, cost hierarchy, and throttle tracking.

### OpenRouter Over Direct API Calls
- Single API key for multiple providers
- Access to free models from various providers
- Tool-calling support built-in
- No need to manage multiple API keys

### Cost Hierarchy (Free First)
- **DEFAULT_MODEL**: `deepseek/deepseek-v4-flash:free` (locked)
- **DEEP_MODEL**: `deepseek/deepseek-chat` (used with `/think` command)
- **CLAUDE_MODEL**: `anthropic/claude-sonnet-4` (used with `/claude` command)
- **Fallback**: On 402 (payment required), fall back to DEFAULT_MODEL

**Rationale:** Account has $0 paid credits. Free models are the only viable option. Paid models are only used when explicitly requested via commands.

### Dynamic Discovery Over Static List
- `model_manager.py` fetches free, tool-capable models from OpenRouter
- Caches model list for 24 hours
- Filters for models that support function calling
- Allows automatic discovery of new free models

**Rationale:** Free model availability changes. Static list would require manual updates. Dynamic discovery ensures we always have access to available free models.

### Throttle Tracking in SQLite
- Table: `model_status` tracks 429 cooldowns per model
- Fields: `model_id`, `throttled_until`, `last_success`, `failure_count`
- On 429: record 5-minute cooldown, skip this model in rotation
- On success: record timestamp, reset failure count
- On 402: record payment required, skip permanently

**Rationale:** 429 errors are frequent. Without tracking, we'd retry throttled models immediately. SQLite persists cooldowns across restarts.

### Fallback Rotation Pattern
- `FREE_MODEL_FALLBACKS` list in agent.py
- On 429: skip to next model in list
- On 402: fall back to DEFAULT_MODEL
- On model unavailable: skip to next model
- Rotation continues until a working model is found

**Rationale:** Free models are unreliable. Single model would fail frequently. Rotation ensures we always find a working model.

## Consequences
### Positive
- Always have a working model (rotation through free models)
- Automatic discovery of new free models
- Persistent throttle tracking across restarts
- Cost-effective (free models only)
- Flexible model selection for different use cases

### Negative
- Free models are lower quality than paid models
- 429 errors cause delays during rotation
- Dynamic discovery adds startup latency (OpenRouter API call)
- Throttle tracking adds database complexity

### Mitigations
- Cache model list for 24 hours to reduce startup latency
- Use 5-minute cooldown for 429 (not permanent)
- Provide `/models` command to show current status
- Allow manual model selection via `/think` and `/claude`

## Alternatives Considered
**Hardcoded single model:** Rejected due to 429 failures and model unavailability.

**Static model list:** Rejected because free model availability changes frequently.

**Paid models only:** Rejected because account has $0 credits.

**No throttle tracking:** Rejected because we'd retry throttled models immediately, causing more 429s.
