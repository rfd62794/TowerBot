# ADR-002: Tool Registry Pattern

## Status
Accepted

## Context
PrivyBot needs a way to expose tools to the AI agent for function calling. The agent needs:
- A list of available tools with their names and descriptions
- Function references to call when the agent selects a tool
- Parameter schemas for OpenAI-compatible function calling

Initial approach used a simple dictionary in `tools/__init__.py`, but this needs formalization as the tool count grows.

## Decision
Use a centralized `TOOL_REGISTRY` dictionary in `tools/__init__.py` as the single source of truth for tool discovery.

### Registry Structure
```python
TOOL_REGISTRY = {
    "tool_name": {
        "fn": function_reference,
        "definition": {
            "type": "function",
            "function": {
                "name": "tool_name",
                "description": "Human-readable description",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "param_name": {
                            "type": "string|integer|array",
                            "description": "Parameter description"
                        }
                    },
                    "required": ["param_name"]
                }
            }
        }
    }
}
```

### Key Principles

**1. Single Source of Truth**
- `TOOL_REGISTRY` is the only place tools are registered
- Agent discovers tools by reading this registry
- No tool is available unless registered here

**2. Separation of Concerns**
- `tools/__init__.py` only contains imports and `TOOL_REGISTRY`
- No function definitions in `__init__.py`
- No API imports in `__init__.py`
- No business logic in `__init__.py`

**3. OpenAI Compatibility**
- `definition` field matches OpenAI function calling schema
- Enables direct use with OpenRouter and other OpenAI-compatible APIs
- Agent can parse definitions without transformation

**4. Function Reference Pattern**
- `fn` field holds direct function reference (not string)
- Enables agent to call: `TOOL_REGISTRY["tool_name"]["fn"](**params)`
- No dynamic imports or string-based function resolution

### Registration Workflow

**Adding a new tool:**
1. Implement function in `tools/tools/` (e.g., `game_tools.py`)
2. Import function in `tools/__init__.py`
3. Add entry to `TOOL_REGISTRY` with name, function reference, and definition
4. Tool is immediately available to agent

**Removing a tool:**
1. Remove entry from `TOOL_REGISTRY`
2. Optionally remove import (kept for future use)
3. Tool is no longer available to agent

### Agent Integration

**Discovery:**
```python
# agent.py
from tools import TOOL_REGISTRY

def get_available_tools():
    """Return tool definitions for OpenAI function calling."""
    return [tool["definition"] for tool in TOOL_REGISTRY.values()]
```

**Execution:**
```python
# agent.py
def call_tool(tool_name: str, params: dict):
    """Execute a tool by name."""
    tool = TOOL_REGISTRY.get(tool_name)
    if not tool:
        return {"error": f"Tool not found: {tool_name}"}
    return tool["fn"](**params)
```

## Consequences
### Positive
- Centralized tool discovery — agent has single place to look
- OpenAI-compatible — no transformation needed for function calling
- Type-safe — function references checked at import time
- Easy to audit — all tools visible in one file
- Simple to add/remove — single edit per tool

### Negative
- Single file bottleneck — all registrations in one place
- Manual registration — must remember to add to registry after implementing
- Import overhead — all tool modules imported at startup

### Mitigations
- Keep registry organized with comments grouping related tools
- Use clear naming convention (tool name matches function name)
- Consider splitting registry by domain if it grows too large
- Lazy imports possible if startup time becomes issue

## Examples

### Complete Registration Example
```python
# tools/__init__.py
from tools.tools.game_tools import get_game_metrics
from tools.tools.channel_tools import get_channel_summary

TOOL_REGISTRY = {
    "get_game_metrics": {
        "fn": get_game_metrics,
        "definition": {
            "type": "function",
            "function": {
                "name": "get_game_metrics",
                "description": "Get detailed metrics for a specific game by name",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "game_name": {
                            "type": "string",
                            "description": "Name of the game"
                        }
                    },
                    "required": ["game_name"]
                }
            }
        }
    },
    "get_channel_summary": {
        "fn": get_channel_summary,
        "definition": {
            "type": "function",
            "function": {
                "name": "get_channel_summary",
                "description": "Get YouTube channel performance for last N days",
                "parameters": {
                    "type": "object",
                    "properties": {
                        "days": {
                            "type": "integer",
                            "description": "Number of days to look back",
                            "default": 7
                        }
                    },
                    "required": []
                }
            }
        }
    }
}
```

### Agent Usage Example
```python
# agent.py
from tools import TOOL_REGISTRY

# Get tool definitions for OpenAI
tools_definitions = [t["definition"] for t in TOOL_REGISTRY.values()]

# Call a tool
result = TOOL_REGISTRY["get_game_metrics"]["fn"](game_name="Raccoin")
```
