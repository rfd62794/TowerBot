# ADR-018: Offline-First Cache Strategy

## Status
Accepted

## Context
PrivyBot makes frequent API calls to external services (YouTube, Steam, Gmail, etc.). Current issues:
- API outages cause bot failures (no fallback)
- First-query latency is high (no preloading)
- No visibility into data staleness
- API quota usage is not optimized

The goal is to make PrivyBot resilient to API failures while maintaining data freshness where it matters.

## Decision
Implement a three-phase offline-first cache strategy with stale fallback and preloading.

### Phase 1: Cache Plumbing (Complete)
Add infrastructure without changing behavior:
- `get_stale_cached_result()` — returns cached data regardless of TTL expiry
- `record_preload_result()` — writes to tool_cache + preload_log
- `get_preload_status()` — metadata on last fetch per tool
- `preload_log` table — tracks fetch attempts, success, duration
- `tools/api/_base.py` — `cached_api_call()` wrapper pattern
- `stale_notice()` — human-readable staleness formatting

### Phase 2: API Wiring (Pending)
Wire all API files to use `cached_api_call()`:
```python
# tools/api/weather_api.py (example)
from tools.api._base import cached_api_call, make_params_hash

def get_current_weather(location: str) -> dict:
    """Get current weather with stale fallback."""
    params_hash = make_params_hash(location=location)

    def live_fn():
        # Existing API call logic
        response = httpx.get(f"{API_BASE}/current", params={"q": location})
        return response.json()

    return cached_api_call(
        tool_name="weather_api",
        params_hash=params_hash,
        live_fn=live_fn,
        ttl_seconds=3600,  # 1 hour
        stale_ok=True,
    )
```

### Phase 3: Preloader (Pending)
Warm cache on bot startup:
```python
# scripts/preload.py
PRELOAD_TASKS = [
    ("weather_api", {"location": "Toronto"}, 3600),
    ("youtube_api", {"days": 7}, 21600),
    # ... more tasks
]

def run_preload():
    """Warm cache on startup."""
    for tool_name, params, ttl in PRELOAD_TASKS:

```

### Staleness Metadata
All cached results include metadata:
```python
{
    "_stale": False,           # True if data is beyond TTL
    "_cached_at": "2026-05-30 09:15:00",
    "_age_minutes": 45,
    # ... actual data
}
```

### Stale Notice Format
Human-readable staleness warnings:
```python
from tools.api._base import stale_notice

notice = stale_notice(result)
# Returns: "⚠️ Data from 4h ago (2026-05-30 09:15)"
# Returns: None if data is fresh
```

### Morning Briefing Stale Cap
To avoid briefing spam:
- Max 2 individual stale notices
- Additional stale data collapsed to summary: "⚠️ 5 tools serving stale data"

## Consequences
### Positive
- Bot remains functional during API outages
- First-query latency reduced via preloading
- User informed when data is stale
- Reduced API quota usage
- Better resilience to rate limits

### Negative
- Stale data may be misleading if not labeled clearly
- Preloader adds startup time
- Cache invalidation complexity
- Morning briefing may show too many stale notices

### Mitigations
- Clear staleness metadata in all responses
- Stale notice format with emoji and timestamp
- Morning briefing stale notice cap (max 2 individual)
- Preloader timeout (skip if > 30 seconds)
- Per-API staleness budgets (see ADR-019)

## Examples

### Fresh cache hit
```python
result = cached_api_call("weather_api", "hash123", live_fn, 3600)
# Returns: {"temp_f": 72, "_stale": False, "_cached_at": "...", "_age_minutes": 5}
```

### Stale fallback on API failure
```python
# API call fails, returns stale data
result = cached_api_call("weather_api", "hash123", failing_fn, 3600)
# Returns: {"temp_f": 70, "_stale": True, "_cached_at": "...", "_age_minutes": 90}
```

### No cache, API failure
```python
# No cached data, API fails
result = cached_api_call("weather_api", "hash123", failing_fn, 3600)
# Returns: {"error": "API timeout", "_live_failed": True, "_stale": False}
```

### Stale notice in tool response
```python
from tools.api._base import stale_notice

result = get_current_weather("Toronto")
if result.get("_stale"):
    result["stale_notice"] = stale_notice(result)
    # "⚠️ Data from 4h ago (2026-05-30 09:15)"
```
